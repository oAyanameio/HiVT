# 实验问题阶段性解决方案文档

## 1. 文档目的

本文档只记录当前实验主线和接下来要做的实验，不再保留已经解决的问题、历史分歧和旧路线。

当前唯一主线是：

```text
freeze-backbone + conservative reranking rule + decision-oriented objective re-eval
```

## 2. 当前结论

截至 2026-06-30，当前可以直接作为实验前提使用的结论只有这些：

1. 默认入口使用 `freeze-backbone`。
   - 不再把 joint training 当主入口。
   - 当前重点是验证 risk head 本身能否改善 mode selection。

2. `mode_risk` 相对宿主原始 `pi` 有真实增量。
   - naive `pi` 风险代理约为 `AUROC=0.70`、`AUPRC=0.71`
   - `mode_risk` 已达到约 `AUROC=0.87`、`AUPRC=0.84`

3. `mode_risk` 已具备较强的 FDE 排序能力。
   - 与真实 `FDE` 的 `Spearman = 0.7482`

4. 概率空间 reranking 已经具备通过保守规则改善 `MR` 的能力。
   - 当前默认保守口径已收敛为 `prob_product + alpha=1.0 + top_k=3 + margin=0.15 + guard=0.0`
   - 在两个独立、干净的 freeze-backbone checkpoint 上，该口径都满足 `reranked_MR < original_MR` 且 `hit_to_miss = 0`

5. 当前核心卡点已经进一步收敛到 reranking rule。
   - 问题不再是“risk score 没学到”
   - 而是“当前无门控 reranking rule 太激进，会把大量低 gap 翻转错误放行”

6. `rerank_margin` 是当前主控制量，`rerank_guard` 暂未显示明显增益。
   - 在 `guard = 0.0 / 0.1 / 0.2 / 0.3` 上结果几乎不变
   - 当前 checkpoint 上决定性因素是 `risk gap` 是否足够大

7. `scene` 和 `conflict` 都不是当前主实验入口。
   - `scene head` 仍过置信，`scene_ECE ≈ 0.236`
   - `conflict` 暂时降优先级，不继续扩展大矩阵分析

8. 当前方向本身没有偏离。
   - `freeze-backbone -> mode_risk -> reranking -> MR` 这条主链仍然成立；
   - 当前的问题不是换方向，而是先把默认决策规则锁稳，再判断训练目标还能否继续增益。

### 3.3 reranking 评估与默认决策口径

基础复评口径统一使用：

```text

```

用途：

- 保留“纯公式、无门控”的历史可比口径；
- 用于判断 checkpoint 本身在激进翻转规则下会不会净造 miss。

当前默认保守决策规则为：

```text

rerank_alpha   = 1.0
rerank_top_k   = 3
rerank_margin  = 0.15
rerank_guard   = 0.0

```

用途：

- 作为当前主线下的默认 reranking 决策逻辑；
- 后续所有新 checkpoint 都应同时报告“基础复评口径”和“保守决策口径”，避免把 rule 问题和 risk head 问题混在一起。

默认值更新说明（2026-07-01）：

- 此前 `margin=0.1` 曾是单个 checkpoint 上的最优点，但未能在独立 freeze-backbone checkpoint（`seed=3407`）上复现 `hit_to_miss=0`；
- 新增的第二个独立干净 freeze-backbone checkpoint（`seed=9103`，配置见 8.1 节）对 `margin=0.10/0.15/0.20` 的 scan 显示：
  - `margin=0.10`：`reranked_MR = 0.421875 < original_MR = 0.425781`，`hit_to_miss=0`，`miss_to_hit=1`；
  - `margin=0.15`：`reranked_MR = 0.421875 < original_MR = 0.425781`，`hit_to_miss=0`，`miss_to_hit=1`；
  - `margin=0.20`：`reranked_MR = 0.421875 < original_MR = 0.425781`，`hit_to_miss=0`，`miss_to_hit=1`；
- 因此 `margin=0.15` 是目前唯一在两个独立、干净的 freeze-backbone checkpoint（`seed=3407` 与 `seed=9103`）上都稳定满足 `reranked_MR < original_MR`、`hit_to_miss=0`、`miss_to_hit>=1` 的候选，已达到 7.3 节定义的稳定性门槛，正式升级为默认 margin；
- `margin=0.1` 降级为“次选/历史参考点”，不再作为默认值；
- `margin=0.20` 仍可作为参考上界观察，但由于 mixed-objective checkpoint 上更接近“收益归零”，当前默认候选仍应收紧在 `margin=0.15`。

## 4. 当前已实现但尚未验证有效的目标改进

本轮已经接入两类目标侧改动，后续实验直接在此基础上继续：

1. 阈值附近样本加权 BCE

```text
mode_risk_threshold_weight_enabled
mode_risk_threshold_weight_radius
mode_risk_threshold_weight_peak
mode_risk_threshold_weight_base
```

作用：

- 提高 `FDE≈2.0m` 附近样本的监督权重；
- 目标是让风险头对是否跨过 MR 阈值更敏感。

2. `top-k + near-threshold` 过滤的 pairwise rank loss

```text
mode_risk_rank_top_k
mode_risk_rank_near_threshold_only
mode_risk_rank_threshold_radius
```

作用：

- 只在更可能影响 top-1 决策的 mode 对上施加排序约束；
- 减少与当前 reranking 目标无关的远离阈值样本干扰。

3. `rank loss` 监督已从 `ADE` 切到 `FDE`

作用：

- 让排序目标和 `MR` / reranking 目标更一致。

## 6. 当前未解决问题

当前只保留一个主问题：

```text
如何把已经验证有效的 conservative reranking rule
稳定固化成默认决策逻辑，并在此基础上判断训练目标是否还能继续带来 MR 增益？
```

当前更具体的判断是：

1. 当前第一瓶颈已经确认在决策规则，而不是 risk head 是否具备全局 FDE 排序能力。
2. 需要优先保护“原 top-1 已经是 hit”的样本，再去开放真正高把握的替代 mode 翻转。
3. 当前 `mode_target = 1(FDE_k > 2.0m)` 仍然是绝对分类目标，而 reranking 真正要解决的是：

```text
在目标车 top-k 候选里，谁应该保住 top-1，谁应该被替换？
```

这意味着当前训练目标与最终 reranking 任务之间仍有结构性错位。
4. 未来训练目标是否需要改写，必须在 `margin=0.15` 这类保守规则下重新判断；否则 rule 误差会继续污染 objective 结论。
5. `near-threshold` 加权 BCE 继续把权重压在 `FDE≈2.0m` 的硬阈值边界上，仍可能放大标签噪声；在新 rule 已成立后，它不再是当前第一优先级。
6. `rerank_guard` 目前不是主要矛盾；只有当后续 checkpoint 出现“大 gap 但 orig_top1 本身风险极低”的异常结构，再考虑重新扩展 guard。

**2026-07-01 更新：`margin=0.15` 已在两个独立种子（`seed=3407` / `seed=9103`）上复现稳定收益，正式替代 `margin=0.1` 作为默认保守决策口径。** 详见本节末尾更新记录。

至少应覆盖：

1. 当前 freeze-backbone 参考 checkpoint；
2. 当前 `top-k only` 小预算 checkpoint；
3. 下一轮中等预算 checkpoint。

目标不是立刻追更低 `MR`，而是先确认：

```text
margin=0.15 的非退化收益是不是稳定存在，而不是当前 checkpoint 的偶然值
```

但这里要明确区分“趋势”与“稳定结论”：

- 当前列入复评的第二个 checkpoint
  `reliability_20260629_191045_dim64_gpu0`
  本身已经混入了新 objective（`rank loss + threshold weighting`）；
- 因此它不能被当成 `rule` 通用性的独立 freeze-backbone 样本；
- 它更适合回答的是：

```text
当 rule 与当前 objective 叠加时，表现会不会更差或更好？
```

而不是单独回答：

```text
rule 本身是否已经稳定成立？
```

如果跨 checkpoint 仍稳定成立，则：

1. 后续所有 objective 复验都必须同时报告“基础复评口径”和“保守决策口径”；
2. 当前默认解决方案就不再是“继续先改训练”，而是“先换保守 rule，再看训练还有没有额外增益”。

这里“稳定成立”按更保守的量化标准理解：

```text
至少 2 个相互独立的 freeze-backbone checkpoint 上，
同一个固定 margin 值满足：
1. reranked_MR <= original_MR
2. hit_to_miss = 0
3. miss_to_hit >= 1
```

**2026-07-01 更新：`margin=0.15` 已满足此标准。** 新增的第二个纯 freeze-backbone 独立种子 `seed=9103`（配置与 `seed=3407` 一致，不含新 objective）复评结果：

```text
margin=0.10 -> reranked_MR=0.421875, hit_to_miss=0, miss_to_hit=1  （seed=3407 上 hit_to_miss=1，不达标）
margin=0.15 -> reranked_MR=0.421875, hit_to_miss=0, miss_to_hit=1  （seed=3407 上同样达标）
margin=0.20 -> reranked_MR=0.421875, hit_to_miss=0, miss_to_hit=1  （seed=3407 上同样达标，但 mixed-objective checkpoint 上更接近收益归零）
```

`margin=0.15` 是当前唯一在两个独立种子上都满足全部三条标准的 margin 值，因此：

```text
rule 的保守门控方向已稳定成立；
默认 margin 从 0.1 升级为 0.15；
后续所有复评统一使用 margin=0.15 作为保守决策口径，
margin=0.20 仅作为收益可能归零的参考上界继续观察，不作为默认候选。
```

`margin=0.20` 在两个独立纯 freeze-backbone 种子上也都满足三条稳定性标准，但由于它在 mixed-objective checkpoint 上更接近“收益归零”的边界，所以当前仍更适合作为参考上界，而不是与 `0.15` 同等级并列的默认候选。

### 7.4 第二优先级：在保守 rule 下重新评估当前 objective 还有没有额外收益

当前 `top-k only` 小预算 run 的负信号，首先是在 `margin=0.0` 的激进 rule 下观察到的。

因此下一步要重新回答的是：

```text
在 margin=0.15 已经锁住坏翻转后，
当前 objective 能不能再把 miss->hit 数量往上推？
```

判读规则：

- 如果新 checkpoint 在 `margin=0.15` 下相对原始 `pi` 有稳定额外收益：
  - 说明 rule 与 objective 可以分层协同；
  - 当前路线继续成立。
- 如果保守 rule 已经吃掉大部分收益，而新 objective 仍然没有额外提升：
  - 说明当前 objective 不是主增益来源；
  - 后续再考虑更直接的 decision-oriented 监督。

**2026-07-02 更新：中等预算 objective re-eval 已完成，但当前 objective 暂未显示超出 baseline 的额外 MR 增量。**

本轮实际运行配置：


在同样的 `64 val batches` 口径下，对照 reference freeze-backbone checkpoint 的结果如下：

2. 中等预算 mixed-objective checkpoint

```text
margin=0.15 -> original_MR=0.449219, reranked_MR=0.447266, hit_to_miss=0,  miss_to_hit=1
```

当前可写结论：

1. `margin=0.15` 这条保守 rule 在中等预算 mixed-objective checkpoint 上继续成立；
2. 但把它和 reference checkpoint 放到同一个 `64 val batches` 口径下比较时，`margin=0.15` 的 `reranked_MR` 同样都是 `0.447266`，`hit_to_miss / miss_to_hit` 也没有额外增量；
3. 因此这轮中等预算 run 只能支持：

```text

但尚未证明它能在已锁稳的 rule 之上继续提供额外 MR 收益。
```

4. 由于这仍然只是单个中等预算 seed，当前更适合把它记为“未观察到额外增量”，而不是“论文级彻底证伪”。
5. 这里还要额外保留一个诊断层面的可能性：

```text
margin=0.15 可能已经过于保守，
把大部分潜在差异都过滤掉了；
因此当前“无增量”不必然等价于“objective 真无效”。
```

6. 当前这一点尤其值得注意，因为在 `64 val batches = 512 cases` 的口径下，
   `margin=0.15` 两边都只保留了极少数真实翻转：
   - reference: `top1_change_rate = 0.007812`，约等于 `4 / 512`
   - mixed-objective: `top1_change_rate = 0.003906`，约等于 `2 / 512`
   - 两边最后都只留下 `miss_to_hit = 1`

这意味着当前主结论仍然成立，但要补一句更精确的判读边界：

```text
当前没有观察到在 margin=0.15 下的额外 MR 增量；
但这也可能部分来自 conservative rule 的 ceiling effect，
而不只是 objective 本身没有带来差异。
```

### 7.5 第三优先级：若保守 rule 稳定后训练仍无额外收益，再改成更直接的 reranking-oriented objective

如果 7.3 / 7.4 之后仍然确认当前 objective 不能再提供增量，则当前路线应从：

```text
near-threshold FDE shaping
```

转向更直接的：

```text
top1-vs-best_alt / miss-oriented objective
```

优先考虑的不是“更大范围的全 mode 排序”，而是更贴近 reranking 场景、且带方向性的监督：

```text
对目标车，只监督 current_top1_mode 与 best_alternative_mode 的竞争关系
```

核心原则是：

- 如果当前 top-1 已经是 hit，优先保护它，不再对称开放坏翻转；
- 如果当前 top-1 是 miss 且存在明显更低风险替代 mode，再增强“允许翻转”的监督；
- `near-threshold BCE` 不再默认继续加大，应先关闭或把 `peak` 降回接近 `1.x`；
- `guard` 继续保持低优先级，除非后续 checkpoint 显示 margin 之外仍有系统性漏保问题。

### 7.6 现在就该做什么：四步执行清单

为了避免继续把 `rule` 问题和 `objective` 问题混在一起，接下来直接按下面四步推进：

     - 基础复评口径：`prob_product + alpha=1.0 + top_k=3 + margin=0.0 + guard=0.0`

2. 先完成跨 checkpoint 的 rule 稳定性复验。
   - 至少复评：
     - `freeze_backbone_budget128_targetmode067`

   - 目标不是继续刷更低 `MR`，而是确认：


当前这一步已经得到新的、更精确的答案：

- `margin=0.1` 本身还不够稳定；
- `margin=0.15` 是当前最强候选；
- `margin=0.20` 更适合作为“收益可能归零”的上界参考；
- 因此接下来的 rule 复验默认不应只盯着 `0.1`，而应至少同时跟踪：

```text

```

3. 中等预算的 objective re-eval 已完成。
   - 本文档把“中等预算”固定定义为：

```text
max_epochs          = 1
limit_train_batches = 256
limit_val_batches   = 64
train_batch_size    = 8
val_batch_size      = 8
```

   - 实际执行的训练入口保持为 `freeze-backbone + current threshold/top-k objective`，并继续沿用 mixed-objective 小预算 run 的配置，只放大预算，不再额外切换 `scene_target_policy`。
   - 这轮预算只用于回答：

```text
链路是否打通
+ 方向是否继续成立
```

   - 不用于单次给出论文级 A/B 结论；如果结果处在边界，至少还要再补一次复验或换 seed。
   - 本轮实际结果是：`margin=0.15` 下与 reference checkpoint 得到相同的 `reranked_MR = 0.447266`，因此当前 objective 暂未显示超出 baseline 的额外 MR 增量。

4. 下一步先补一个第二个 mixed-objective seed 复验，再决定是否切换到更直接的 supervision。
   - 只有当下面两件事同时成立时，才进入 `top1-vs-best_alt / miss-oriented objective`：

```text
conservative rule 已稳定成立
+ 当前 objective 在 conservative rule 下仍无额外收益
```

   - 当前第一条已经满足；
   - 第二条在单个中等预算 run 上已经出现，但为了避免把单次边界结果写成过强结论，下一步更合适的是：

```text
先补 1 个 mixed-objective 的独立 seed / 复跑；
并把诊断性评估从单点 margin=0.15
扩成 margin = 0.05 / 0.10 / 0.15；
若在这组 conservative margin 下仍无额外 MR 增量或趋势，
再正式转入 top1-vs-best_alt / miss-oriented objective。
```

   - 其中判读优先级应明确分成两层：

```text
主结论层：
  margin=0.15 下是否出现稳定额外 MR 增量

诊断层：
  margin=0.05 / 0.10 下是否出现更高的 miss_to_hit 趋势，
  以区分“objective 真无效”与“rule ceiling 掩盖差异”
```

执行前额外注意：

- 继续开新 run 前先确认 `/home` 空间，避免再次因为 checkpoint 保存失败污染结论；
- 后续正文中不再把 `margin=0.0` 下的负信号直接写成 objective 的最终结论。
- `guard=0.0` 在当前实现下近似空操作，当前扫描结果里它几乎不影响结论，更像结构性结果而不只是当前 checkpoint 的偶然现象。
