# 实验问题阶段性解决方案文档

## 1. 文档目的

本文档只记录当前实验主线和接下来要做的实验，不再保留已经解决的问题、历史分歧和旧路线。

当前唯一主线是：

```text
freeze-backbone + conservative reranking rule + decision-oriented objective re-eval
```

## 2. 当前结论

截至 2026-06-30，当前可以直接作为实验前提使用的结论只有这些：

1. 默认入口使用 `freeze-backbone`。
   - 不再把 joint training 当主入口。
   - 当前重点是验证 risk head 本身能否改善 mode selection。

2. `mode_risk` 相对宿主原始 `pi` 有真实增量。
   - naive `pi` 风险代理约为 `AUROC=0.70`、`AUPRC=0.71`
   - `mode_risk` 已达到约 `AUROC=0.87`、`AUPRC=0.84`

3. `mode_risk` 已具备较强的 FDE 排序能力。
   - 与真实 `FDE` 的 `Spearman = 0.7482`

4. 概率空间 reranking 已经具备通过保守规则改善 `MR` 的能力。
   - 当前 `prob_product` 公式仍然可以改善 `minADE / minFDE`
   - 在当前小预算 checkpoint 上，加入 `rerank_margin=0.1` 后已得到 `reranked_MR = 0.421875 < original_MR = 0.425781`

5. 当前核心卡点已经进一步收敛到 reranking rule。
   - 问题不再是“risk score 没学到”
   - 而是“当前无门控 reranking rule 太激进，会把大量低 gap 翻转错误放行”

6. `rerank_margin` 是当前主控制量，`rerank_guard` 暂未显示明显增益。
   - 在 `guard = 0.0 / 0.1 / 0.2 / 0.3` 上结果几乎不变
   - 当前 checkpoint 上决定性因素是 `risk gap` 是否足够大

7. `scene` 和 `conflict` 都不是当前主实验入口。
   - `scene head` 仍过置信，`scene_ECE ≈ 0.236`
   - `conflict` 暂时降优先级，不继续扩展大矩阵分析

8. 当前方向本身没有偏离。
   - `freeze-backbone -> mode_risk -> reranking -> MR` 这条主链仍然成立；
   - 当前的问题不是换方向，而是先把默认决策规则锁稳，再判断训练目标还能否继续增益。

## 3. 当前默认实验口径

### 3.1 训练入口

当前训练入口固定为：

```text
warm start from freeze-backbone reference checkpoint
+ freeze backbone
+ only optimize reliability-side objective
```

参考 checkpoint：

```text

```

### 3.2 mode_risk 主监督

当前主监督仍然使用：

```text
mode_target_policy = fde_only
risk_fde_threshold = 2.0
```

含义：

- 先围绕 `FDE=2.0m` 这一决策边界提升分辨率；
- 暂不把 `miss_targets` 重新并回主标签；
- `MR` 作为 reranking 复验指标，而不是先改成新的主监督定义。

### 3.3 reranking 评估与默认决策口径

基础复评口径统一使用：

```text
method         = prob_product
rerank_alpha   = 1.0
rerank_top_k   = 3
rerank_margin  = 0.0
rerank_guard   = 0.0
eval_batches   = 32 val batches
```

用途：

- 保留“纯公式、无门控”的历史可比口径；
- 用于判断 checkpoint 本身在激进翻转规则下会不会净造 miss。

当前默认保守决策规则为：

```text
method         = prob_product
rerank_alpha   = 1.0
rerank_top_k   = 3
rerank_margin  = 0.1
rerank_guard   = 0.0
eval_batches   = 32 val batches
```

用途：

- 作为当前主线下的默认 reranking 决策逻辑；
- 后续所有新 checkpoint 都应同时报告“基础复评口径”和“保守决策口径”，避免把 rule 问题和 risk head 问题混在一起。

## 4. 当前已实现但尚未验证有效的目标改进

本轮已经接入两类目标侧改动，后续实验直接在此基础上继续：

1. 阈值附近样本加权 BCE

```text
mode_risk_threshold_weight_enabled
mode_risk_threshold_weight_radius
mode_risk_threshold_weight_peak
mode_risk_threshold_weight_base
```

作用：

- 提高 `FDE≈2.0m` 附近样本的监督权重；
- 目标是让风险头对是否跨过 MR 阈值更敏感。

2. `top-k + near-threshold` 过滤的 pairwise rank loss

```text
mode_risk_rank_top_k
mode_risk_rank_near_threshold_only
mode_risk_rank_threshold_radius
```

作用：

- 只在更可能影响 top-1 决策的 mode 对上施加排序约束；
- 减少与当前 reranking 目标无关的远离阈值样本干扰。

3. `rank loss` 监督已从 `ADE` 切到 `FDE`

作用：

- 让排序目标和 `MR` / reranking 目标更一致。

### 5.3 当前解释

1. 目标函数改动已经真实进入训练与 checkpoint。
2. 这轮 `smoke` 不具备评估机制有效性的能力：
   - `1 epoch / 2 train batches / 2 val batches` 只能用于检查链路是否打通；
   - 不能据此判断 near-threshold objective 是否有效或无效。
3. 训练后对新 checkpoint 的 reranking 复评仍然给出了一个弱负信号：
   - 当前 `MR` 没有改善；
   - 但这只能说明“undertrained checkpoint 尚未显示收益”，不能直接推出“objective 无效”。
4. 因此当前结论只能是：

```text
near-threshold objective 已实现、可继续调，但尚未证明有效
```


随后按固定口径做 reranking 复评（`prob_product + rerank_alpha=1.0 + rerank_top_k=3 + 32 val batches`），结果为：

```text
original_minADE = 1.198633

reranked_minADE = 1.195999
```

当前解释：

1. 这轮结果不只是“undertrained 尚未显示收益”，而是一个轻度负信号：
   - `hit -> miss = 14`
   - `miss -> hit = 3`
   - 相比旧 freeze-backbone 参考结果（`9 -> 3`），方向进一步变差。
2. 当前 reranking 仍然主要带来连续误差上的轻微收益，而不是 miss 决策收益；
3. 这说明即使把 rank loss 从 `strict_current` 放宽到 `top-k only`，当前 objective 仍然没有闭合 “risk -> top-1 -> MR” 这一段；
4. 更关键的是：当前路线主要仍在继续强化“按 FDE 排序”，但 `MR` 的主瓶颈很可能已经转移到 reranking 决策规则本身，而不是全局 FDE 排序能力。

因此，下一步不应再继续把当前 objective 当成默认有效方案；更优先的动作是先做零训练成本的 reranking rule 反事实扫描，再决定是否需要继续改训练目标。

### 5.5 最新结果：reranking rule 反事实扫描与 case 复盘

在上述 `top-k only` 小预算 checkpoint 上，按固定口径新增扫描：

```text
method         = prob_product
rerank_alpha   = 1.0
rerank_top_k   = 3
eval_batches   = 32 val batches
margin_grid    = 0.0 / 0.05 / 0.10 / 0.15 / 0.20
guard_grid     = 0.0 / 0.1 / 0.2 / 0.3
```

按 `margin` 聚合后的关键结果如下：

| margin | reranked_MR | top1_change_rate | hit->miss | miss->hit | 当前解释 |
|---:|---:|---:|---:|---:|---|
| `0.00` | `0.468750` | `0.250000` | `14` | `3` | 无门控，明显过激 |
| `0.05` | `0.445312` | `0.121094` | `6` | `1` | 仍然净造 miss |
| `0.10` | `0.421875` | `0.023438` | `0` | `1` | 当前最优，已优于原始 MR |
| `0.15` | `0.421875` | `0.011719` | `0` | `1` | 更保守，但无额外收益 |
| `0.20` | `0.425781` | `0.000000` | `0` | `0` | 等价于不 rerank |

额外结论：

1. `guard = 0.0 / 0.1 / 0.2 / 0.3` 在当前 checkpoint 上几乎不改变结果；
2. 当前真正起作用的是 `margin`，不是 `guard`；
3. 因此 7.1 已经得到明确答案：

```text
存在保守 reranking rule，可以让 reranked_MR <= original_MR
```

进一步把 `margin=0.0` 与 `margin=0.1` 做逐 case 对比：

```text
original_MR              = 0.425781
base_MR (margin=0.0)     = 0.468750
compare_MR (margin=0.1)  = 0.421875

base_switch_count        = 64
compare_switch_count     = 6
blocked_switch_count     = 58

blocked_harmful_count    = 14
blocked_helpful_count    = 2
blocked_neutral_count    = 42
```

这组 case 对比说明：

1. `margin=0.1` 把 `64` 次候选翻转压缩成 `6` 次真实翻转；
2. 它挡掉了全部 `14` 个 `hit -> miss` 坏翻转；
3. 只额外牺牲了 `2` 个原本能变好的 `miss -> hit`；
4. 同时还保住了 `1` 个真正强信号的 `miss -> hit`。

更关键的是 risk gap 分布出现了清晰分离：

```text
blocked_switch_gap_q10/q50/q90 = 0.016 / 0.044 / 0.083
kept_switch_gap_q10/q50/q90    = 0.119 / 0.153 / 0.195
```

这说明在当前 checkpoint 上：

1. 被 `margin=0.1` 保留下来的翻转全部来自明显更大的 risk gap；
2. 被挡住的翻转即使是“好翻转”，gap 也只有 `0.015 ~ 0.018`，明显更像弱信号；
3. 唯一保留下来的 `miss -> hit` case，`risk_gap = 0.194`，`original_fde = 15.908`，`reranked_fde = 1.162`，属于高置信、强收益翻转；
4. 因此当前 `MR` 退化的主要来源不是 risk head 缺乏排序能力，而是无门控 rule 把太多低 gap、低把握翻转也一并放行了。

## 6. 当前未解决问题

当前只保留一个主问题：

```text
如何把已经验证有效的 conservative reranking rule
稳定固化成默认决策逻辑，并在此基础上判断训练目标是否还能继续带来 MR 增益？
```

当前更具体的判断是：

1. 当前第一瓶颈已经确认在决策规则，而不是 risk head 是否具备全局 FDE 排序能力。
2. 需要优先保护“原 top-1 已经是 hit”的样本，再去开放真正高把握的替代 mode 翻转。
3. 当前 `mode_target = 1(FDE_k > 2.0m)` 仍然是绝对分类目标，而 reranking 真正要解决的是：

```text
在目标车 top-k 候选里，谁应该保住 top-1，谁应该被替换？
```

这意味着当前训练目标与最终 reranking 任务之间仍有结构性错位。
4. 未来训练目标是否需要改写，必须在 `margin=0.1` 这类保守规则下重新判断；否则 rule 误差会继续污染 objective 结论。
5. `near-threshold` 加权 BCE 继续把权重压在 `FDE≈2.0m` 的硬阈值边界上，仍可能放大标签噪声；在新 rule 已成立后，它不再是当前第一优先级。
6. `rerank_guard` 目前不是主要矛盾；只有当后续 checkpoint 出现“大 gap 但 orig_top1 本身风险极低”的异常结构，再考虑重新扩展 guard。

## 7. 接下来实验顺序

### 7.1 已完成：零训练成本 reranking rule 扫描

在继续训练前，先在现有 checkpoint 上直接扫描 reranking 决策规则，不再先开新 run。

```text
结论：
在不改风险头参数的前提下，已经找到更保守的翻转规则，
使 reranked_MR <= original_MR
```

当前最优候选为：

```text
method         = prob_product
rerank_alpha   = 1.0
rerank_top_k   = 3
rerank_margin  = 0.1
rerank_guard   = 0.0
```

判读规则：

- `reranked_MR = 0.421875 < original_MR = 0.425781`
- `hit_to_miss = 0`
- `miss_to_hit = 1`
- `rerank_top1_change_rate = 0.023438`

因此，当前主瓶颈已经判定为：

```text
reranking rule 太激进，而不是 risk head 完全分不开决策相关 mode pair
```

### 7.2 已完成：基于规则扫描结果的 case 复盘

本轮已补做：

1. `hit -> miss` / `miss -> hit` 数量变化；
2. `pi_top1` 与 `best_alt` 的 risk gap 分布；
3. 被 `margin=0.1` 挡住的坏翻转 / 好翻转数量；
4. 唯一保留下来的强收益 `miss -> hit` case。

本轮得到的核心解释是：

```text
低 gap 翻转是当前 MR 退化主因；
margin=0.1 已经能把强信号翻转和弱信号翻转清晰分开
```

### 7.3 第一优先级：把 conservative rule 固化成当前默认决策逻辑，并做跨 checkpoint 复验

在接下来所有 freeze-backbone checkpoint 上，统一新增一套保守决策复评：

```text
prob_product + alpha=1.0 + top_k=3 + margin=0.1 + guard=0.0
```

至少应覆盖：

1. 当前 freeze-backbone 参考 checkpoint；
2. 当前 `top-k only` 小预算 checkpoint；
3. 下一轮中等预算 checkpoint。

目标不是立刻追更低 `MR`，而是先确认：

```text
margin=0.1 的非退化收益是不是稳定存在，而不是当前 checkpoint 的偶然值
```

如果跨 checkpoint 仍稳定成立，则：

1. 后续所有 objective 复验都必须同时报告“基础复评口径”和“保守决策口径”；
2. 当前默认解决方案就不再是“继续先改训练”，而是“先换保守 rule，再看训练还有没有额外增益”。

### 7.4 第二优先级：在保守 rule 下重新评估当前 objective 还有没有额外收益

当前 `top-k only` 小预算 run 的负信号，首先是在 `margin=0.0` 的激进 rule 下观察到的。

因此下一步要重新回答的是：

```text
在 margin=0.1 已经锁住坏翻转后，
当前 objective 能不能再把 miss->hit 数量往上推？
```

判读规则：

- 如果新 checkpoint 在 `margin=0.1` 下相对原始 `pi` 有稳定额外收益：
  - 说明 rule 与 objective 可以分层协同；
  - 当前路线继续成立。
- 如果保守 rule 已经吃掉大部分收益，而新 objective 仍然没有额外提升：
  - 说明当前 objective 不是主增益来源；
  - 后续再考虑更直接的 decision-oriented 监督。

### 7.5 第三优先级：若保守 rule 稳定后训练仍无额外收益，再改成更直接的 reranking-oriented objective

如果 7.3 / 7.4 之后仍然确认当前 objective 不能再提供增量，则当前路线应从：

```text
near-threshold FDE shaping
```

转向更直接的：

```text
top1-vs-best_alt / miss-oriented objective
```

优先考虑的不是“更大范围的全 mode 排序”，而是更贴近 reranking 场景、且带方向性的监督：

```text
对目标车，只监督 current_top1_mode 与 best_alternative_mode 的竞争关系
```

核心原则是：

- 如果当前 top-1 已经是 hit，优先保护它，不再对称开放坏翻转；
- 如果当前 top-1 是 miss 且存在明显更低风险替代 mode，再增强“允许翻转”的监督；
- `near-threshold BCE` 不再默认继续加大，应先关闭或把 `peak` 降回接近 `1.x`；
- `guard` 继续保持低优先级，除非后续 checkpoint 显示 margin 之外仍有系统性漏保问题。

## 8. 暂不展开的方向

以下内容不再作为当前主实验展开项：

1. joint training 回炉
2. scene 作为主卖点扩写
3. conflict 的大规模矩阵展开
4. off-road 重新并入主监督

当前处理原则：

- `scene` 只做输入聚合 / 信息瓶颈检查；
- `scene_loss_weight` 可以在 mode-only 对照里进一步下调，避免噪声干扰；
- `conflict` 只保留低优先级辅助分析；
- 主时间预算集中在 `mode_risk -> reranking -> MR` 这条链上。
