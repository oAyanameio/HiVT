### 3. 预测精度指标

#### `val_minADE`

`val_minADE` 不是在所有 actor 上算的，而是**只在每个场景的目标车 `AGENT` 上算**。

当前代码流程是：

1. 用 `data['agent_index']` 取出每个场景的目标车
2. 模型输出 `K=6` 条预测轨迹
3. 先按终点误差最小选出 best mode
4. 再用这条 best mode 计算整条未来轨迹的 ADE
5. 最后在整个验证集所有场景上取平均

当前实现对应的含义是：

```text
best mode = argmin_k FDE_k
ADE = mean_t ||y_pred(t) - y_gt(t)||_2
val_minADE = 验证集所有场景目标车的平均值
```

对应代码：

- [hivt.py](/home/lbh/HiVT/models/hivt.py:340)
- [ade.py](/home/lbh/HiVT/metrics/ade.py:32)

#### `val_minFDE`

同样只在目标车上计算。

流程是先选出终点误差最小的 best mode，再计算这条轨迹最后一个未来时刻的终点误差：

```text
best mode = argmin_k FDE_k
FDE = ||y_pred(T) - y_gt(T)||_2
val_minFDE = 验证集所有场景目标车的平均值
```

对应代码：

- [hivt.py](/home/lbh/HiVT/models/hivt.py:343)
- [fde.py](/home/lbh/HiVT/metrics/fde.py:32)

#### `val_minMR`

同样只在目标车上计算。

当前实现里 miss 的定义是：

也就是 best mode 的终点误差如果大于 `2.0m`，这个样本就记为 miss。

### 4. 可靠性指标

#### `val_mode_AUROC`

这是 **mode 级失败检测** 的 ROC-AUC。

- 样本单位：一个 `(actor, mode)` 对
- 样本范围：所有有效 actor，经 `valid_mask` 过滤
- 分数：预测的 `mode_risk`
- 标签：`mode_targets`

直观上，它表示：

> 随机抽一个正样本 mode 和一个负样本 mode，模型给正样本更高风险分数的概率有多大。

#### `val_mode_AUPRC`

这是同一个 mode 级失败检测任务上的 PR-AUC。

- 样本单位：一个 `(actor, mode)` 对
- 样本范围：所有有效 actor，经 `valid_mask` 过滤

当正样本比例不均衡时，AUPRC 往往比 AUROC 更敏感。

#### `val_mode_BrierScore`

这是预测风险概率与二值失败标签之间的均方误差：

```text

```

- 样本单位：一个 `(actor, mode)` 对
- 样本范围：所有有效 actor，经 `valid_mask` 过滤

越低越好。

对应代码：

#### `val_mode_ECE`

这是 mode 级风险概率的校准误差。

- 样本单位：一个 `(actor, mode)` 对

- 当前实现：`10` 个 bin

当前代码里的定义是：

```text

```

越低越好。

### 0. 宿主原始 `pi` 与 freeze-backbone 诊断补充（2026-06-28）

这一轮补的是之前缺失的三类诊断：

- 宿主原始 `pi` 的 naive reliability baseline
- `mode_risk` 与实际 `FDE` 的排序相关性
- `scene` 级校准指标

同时补做了 freeze-backbone 与 reranking-only 对照。

#### 0.1 宿主 checkpoint 的 naive baseline

使用宿主 `HiVT-64` checkpoint：

```text
/home/lbh/HiVT/checkpoints/HiVT-64/checkpoints/epoch=63-step=411903.ckpt
```

以

```text
risk_proxy_naive = -log softmax(pi_k)
```

作为 mode-level failure risk 代理，对 `FDE > 2.0m` 的标签计算得到：

```text
naive_mode_AUROC = 0.700041
naive_mode_AUPRC = 0.714026
```

这说明宿主原始 `pi` 本身就有一定 failure detection 能力，但明显低于之前 plugin run 中 `mode_risk` 的 `AUROC/AUPRC`。

#### 0.2 Freeze-backbone 结果

正确的 freeze-backbone 设置是：

- 从宿主 `HiVT-64` checkpoint 热启动 backbone
- `freeze_backbone=true`
- 只训练 `reliability_module`

注意：如果不先加载宿主 checkpoint，而是直接冻结随机初始化 backbone，那么预测指标会失真，那种结果不能用于结论。

本轮正确 freeze-backbone run 的验证指标为：

```text
val_minADE = 0.574
val_minFDE = 0.803
val_minMR  = 0.0508
```

与宿主 baseline 基本一致。可用结论是：

1. freeze-backbone 成功避免了联合训练对宿主预测性能的破坏；
2. 之前 `minADE/minFDE/MR` 的退化，主要来自 reliability loss 对 backbone 的梯度干扰，而不是 reliability 分支本身一定无效。

#### 0.3 Freeze-backbone checkpoint 上的风险诊断

对 freeze-backbone 的正确 checkpoint 继续做分析，得到：

```text
naive_mode_AUROC        = 0.695946
naive_mode_AUPRC        = 0.713348
mode_risk_fde_spearman  = 0.748202
scene_BrierScore        = 0.254131
scene_ECE               = 0.235648
```

这里的含义需要分开看：

1. `naive_mode_AUROC/AUPRC` 仍然只是在评估宿主原始 `pi` 的 failure detection 能力；
2. `mode_risk_fde_spearman=0.7482` 说明 `mode_risk` 对真实 `FDE` 已经具备较强排序能力；
3. `scene_BrierScore/ECE` 仍然偏差较大，说明 scene head 目前更像“有粗判别性但未校准”的分支。

#### 0.4 Reranking-only 结果

在同一个 freeze-backbone checkpoint 上，对比 `original pi` 与 `pi - alpha * risk` 的 top-1 重排结果：

| alpha | original minADE | original minFDE | original MR | reranked minADE | reranked minFDE | reranked MR | top1 change rate |
|---:|---:|---:|---:|---:|---:|---:|---:|
| 0.2 | 1.198633 | 2.575924 | 0.425781 | 1.201279 | 2.585675 | 0.437500 | 0.023438 |
| 0.5 | 1.198633 | 2.575924 | 0.425781 | 1.209395 | 2.598438 | 0.445312 | 0.074219 |
| 1.0 | 1.198633 | 2.575923 | 0.425781 | 1.219001 | 2.613258 | 0.449219 | 0.128906 |

可用结论：

1. reranking 确实改变了 top-1 mode，不是“risk 分数完全没起作用”；
2. 当前 `pi - alpha * risk` 公式在 `alpha=0.2/0.5/1.0` 下都会带来小幅退化；
3. 因此当前问题不是 `mode_risk` 没学到，而是“当前重排映射不合适”。

进一步按评审意见补做了概率空间 reranking 对照。当前脚本新增了两类公式：

- `prob_product`: `log_softmax(pi) + alpha * log(1 - risk + eps)`
- `logprob_minus_norm_risk`: `log_softmax(pi) - alpha * normalized_risk`

并支持只在 `top-k` 候选 mode 内重排。

同一 freeze-backbone checkpoint、同一 32 个验证 batch 上的结果如下：

| method | alpha | top-k | original minADE | original minFDE | original MR | reranked minADE | reranked minFDE | reranked MR | top1 change rate |
|---|---:|---:|---:|---:|---:|---:|---:|---:|---:|
| `prob_product` | 0.5 | 3 | 1.198633 | 2.575924 | 0.425781 | 1.172185 | 2.487767 | 0.449219 | 0.214844 |
| `prob_product` | 1.0 | 3 | 1.198633 | 2.575923 | 0.425781 | 1.172185 | 2.487766 | 0.449219 | 0.214844 |
| `logprob_minus_norm_risk` | 0.5 | 3 | 1.198633 | 2.575923 | 0.425781 | 1.207332 | 2.574770 | 0.472656 | 0.324219 |

当前可用结论：

1. 把重排切回概率空间后，`minADE/minFDE` 已不再像旧公式那样稳定退化；
2. 但 `MR` 仍然明显变差，说明“概率语义正确”还不等于“mode selection 已经可直接获益”；
3. 在当前脚本口径下，`prob_product` 的最优重排实际已经落在原始 top-3 内；`alpha=0.5` 与 `1.0` 给出相同 argmax，说明当前更敏感的不是这两个 alpha，而是公式本身；
4. `logprob_minus_norm_risk` 明显更差，`top1 change rate=0.324219`，`MR=0.472656`，说明单纯把标准化 risk 线性扣到 log-prob 上会引入更多坏翻转。

进一步做 case analysis 后，`prob_product + alpha=1.0 + top_k=3` 的 `256` 个验证样本分桶如下：

```text
case_mean_fde_delta          = -0.088157
case_hit_to_miss_count       = 9
case_miss_to_hit_count       = 3
case_still_miss_improved     = 16
case_still_miss_worsened     = 10
case_still_miss_unchanged    = 80
case_still_hit_improved      = 6
case_still_hit_worsened      = 11
case_still_hit_unchanged     = 121
```

这组分桶给出的核心解释是：

1. `MR` 变差的直接原因不是“大量样本被彻底换坏”，而是 `hit->miss = 9` 多于 `miss->hit = 3`，净增 `6` 个 miss；
2. `minFDE` 下降主要来自 `still_miss_improved = 16`，也就是一批“本来就 miss 的样本”被拉近了，但还没有跨过 `2.0m` 的 MR 阈值；
3. 因而当前 reranking 更像是在优化连续的终点误差，而不是稳定优化 `miss` 这个阈值事件；
4. 这也说明后续若还要追求 `MR` 收益，目标不能只继续对齐 `FDE` 排序，还要单独分析阈值附近样本。

## 当前最直接的结论

1. 当前标签修复是有效的：
   - `mode_risk_target_rate` 不再塌到 `1.0`
   - `scene_risk_target_rate` 也不再塌到 `1.0`
   - `conflict_risk_target_rate` 已降到极低水平，没有再污染主监督

2. 当前 mode 级风险预测已经具备可判别性：
   - `AUROC` 达到 `0.85 ~ 0.87`
   - `AUPRC` 达到 `0.80 ~ 0.84`
   - `ECE` 降到 `0.03 ~ 0.07`

3. 宿主原始 `pi` 的 naive baseline 明显低于 reliability 分支：
   - naive `AUROC/AUPRC` 约为 `0.70/0.71`
   - plugin `mode_risk` 在已有 run 中达到 `0.87/0.84`
   - 因此 reliability 分支在 failure detection 上有真实增量，不只是重复宿主 `pi`

4. freeze-backbone 已经验证通过：
   - 它可以基本保住宿主 `minADE / minFDE / MR`
   - 联合训练退化的主要来源是 backbone 受 reliability loss 干扰

5. `mode_risk` 已具备较强 FDE 排序能力：
   - `Spearman(mode_risk, FDE) = 0.7482`
   - 因此 reranking 无收益不能简单归因于 risk 学坏了

6. 当前最需要解决的不是继续堆训练，而是改进 reranking 映射：
   - 当前 `pi - alpha * risk` 会稳定带来小幅退化
   - scene 校准也仍然偏弱，`ECE ≈ 0.236`
